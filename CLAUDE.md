# iOS Project - Claude Context

## Project Overview
Native Swift/SwiftUI iOS app for Prosaurus — a social media platform with chat, blogs, collections, sessions, and employment features. Same backend API as the web app at https://www.prosaurus.com/api.

> **Naming note**: Prosaurus is the current product name. The codebase still uses `Breakroom` at the code level (bundle ID `com.cherryblossomdev.Breakroom`, folder names, etc.) — this is intentional and not a mistake. "Breakroom" now refers specifically to the main social feed page within the app.

## Tech Stack

- Swift 6.0, iOS 17+, SwiftUI
- Socket.IO for real-time chat
- XcodeGen for project generation (`project.yml`)

## Environments

Five environments exist across all clients (Web, Android, iPhone):

| # | Name | Where Hosted | Backend URL | Database |
|---|------|--------------|-------------|----------|
| 1 | **Production** | EC2 | https://www.prosaurus.com | breakroom (EC2) |
| 2 | **Dev** | EC2 | https://dev.prosaurus.com | breakroom-dev (EC2) |
| 3a | **Local/Dev** | Dev machine | http://localhost:3000 | breakroom-dev (EC2) |
| 3b | **Local/Prod** | Dev machine | http://localhost:3000 | breakroom (EC2, production) |
| 4 | **Test** | Dev machine | http://localhost:3001 | breakroom_test (generated, isolated) |

**Notes:**
- **Local/Prod (3b)** is the most commonly used during active development — local backend hitting the production database
- **Local/Dev (3a)** is for testing changes against dev-tier data without touching production
- **Test (4)** uses a freshly seeded, deterministic database so automated tests produce consistent results
- Environments 3a, 3b, and 4 run a local backend on the dev machine; the iOS Simulator reaches it via `localhost`

### Switching Environments

```bash
./switch-env.sh local-prod   # Local backend + production database (most common)
./switch-env.sh local-dev    # Local backend + dev database
./switch-env.sh test         # Local backend + isolated test database
./switch-env.sh dev          # EC2 dev server
./switch-env.sh production   # EC2 production server
```

This updates `Config.swift` with the correct `baseURL`. For local environments (`local-dev`, `local-prod`, `test`), it also restarts the backend in `../Breakroom` with the correct docker-compose and `.env` file. Rebuild after switching.

## Build Commands

```bash
# Generate project from project.yml (required after pulling changes)
xcodegen generate

# Build via command line
xcodebuild -scheme Breakroom -configuration Debug -destination 'platform=iOS Simulator,name=iPhone 17' build
```

## Project Structure

```
Breakroom/
├── Models/          # Data models (Codable structs)
├── Views/           # SwiftUI views organized by feature
├── ViewModels/      # ObservableObject view models
├── Services/        # API services and managers
├── Components/      # Reusable UI components
├── Config.swift     # Environment config (auto-generated by switch-env.sh)
└── BreakroomApp.swift
```

## Architecture

- **MVVM pattern**: Views observe ViewModels, ViewModels call Services
- **Network**: Native `URLSession` via `APIClient` singleton (no third-party HTTP library)
- **Auth**: JWT stored in Keychain via `KeychainManager`
- **Real-time**: Socket.IO (`socket.io-client-swift:16.1.1`) in `SocketManager`
- **Subscriptions**: Native StoreKit via `StoreKitManager`

## Key Files

| File | Purpose |
|------|---------|
| `project.yml` | XcodeGen project definition, dependencies, build settings |
| `Config.swift` | Environment config (auto-generated by switch-env.sh) |
| `Services/APIClient.swift` | Base HTTP client with auth headers, token refresh |
| `Services/AuthService.swift` | Login/logout, session management |
| `Services/KeychainManager.swift` | Secure token storage |
| `Services/SocketManager.swift` | Socket.IO connection and events |
| `Services/StoreKitManager.swift` | In-app purchases and subscriptions |

## Dependencies (Notable)

| Package | Version | Purpose |
|---------|---------|---------|

All other functionality uses native Apple frameworks (URLSession, Keychain, StoreKit, SwiftUI).

## Known Issues / Notes

- `Config.swift` is auto-generated — do not edit manually; use `./switch-env.sh`
- iOS Simulator uses `localhost` to reach local backend (unlike Android which needs `10.0.2.2`)
- Tests require Docker running locally with the test backend (`./switch-env.sh test`)
